Mobile apps killed the world-wide-web; this website is an interpetation of how I anticipate information will be organized in the future.. Maybe not this exact design, but some kind of consistent, directory-based structure, that allows users to drill down to the information they seek, without encountering any new navigational formats, pop-up advertisements, viruses, phising attempts, or any of the other things the world wide web seems to have devolved into including.

I call it GFLOW.. Not nearly finished, a bit rushed, but getting there.. Basically a prototype. I don't know all of these fancy new development techniques, but the idea is to create a universal codebase that can easily be replicated for use by a variety of individuals, for a variety of types of websites.

It's a bit bloated on the script side of things, the design is a bit off, mobile needs work, and I'd love to host the core code in a git repository and create a CMS that allows people to easily create sites of thier own, but like I said, the web is dead, and I'm busy, so it probably won't ever get finished.
